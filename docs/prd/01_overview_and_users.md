# 1. 개요 및 타깃 사용자

[<- 마스터 인덱스로 돌아가기](./index.md)

## 1. 개요 (Overview)

- **프로젝트 이름**: BiveMap (바이브맵) 
- **한 줄 설명**: 흩어진 지식을 자동 수집하고, 시각적 거대 마인드맵과 RAG 챗봇을 결합하여 초심자의 바이브 코딩 작업을 원스톱으로 지원하는 '올인원 조감형 세컨드 브레인'.
- **프로젝트 배경과 목적**: 프롬프트 엔지니어링이나 최신 AI 도구에 대한 정보가 블로그/X/커뮤니티 등 매우 여러 곳에 파편화되어 있음. 기존의 폴더형 노트 수집(Notion 등) 방식은 전체 복잡성을 직관적으로 파악하기 어렵고, 실제 작업 환경과 쉽게 분리되는 맥락 단절의 고충이 존재함. 파편화된 지식을 하나의 전체 지도(Map)로 직관적으로 보여주며(조감), **URL/텍스트만 던지면 비용이나 수고(Friction) 없이 알아서 지식이 맵에 주입 및 구조화되게 만드는 것**이 핵심 목표임.

## 2. 타깃 사용자 (Target User Persona)

**페르소나 1: 이성장 (32세, 서비스 기획자/비개발 직군) - [주요 핵심 타깃]**
- **니즈**: 평소 코딩을 하나도 모르지만 최신 AI(Cursor 등)를 활용한 바이브 코딩으로 내 머릿속 아이디어를 당장 쓸모있는 프로덕트로 구현해보고 싶어 함.
- **고충**: 전문 용어가 나올 때마다 두려움을 느끼며, 웹 서핑 중 꿀팁을 주워 담아도 어디다 두었는지, 언제 써먹는지 기억을 못 함. 방대한 지식을 시각적 지도처럼 한 장병으로 보고 싶어 함.

**페르소나 2: 김효율 (28세, 주니어 프런트엔드 개발자)**
- **니즈**: 단순 코딩을 넘어, 팀 내에서 프롬프트 체이닝을 가장 잘 다루고 에러 해결을 위한 최적의 AI 아키텍처를 빠르게 도입하고 싶음.
- **고충**: 매일 쏟아지는 영문 해외 소스(HackerNews 등)와 기술 동향을 팔로업할 시간이 부족함. 누군가 무조건 알아서 수집해서 텍스트로 요약 정리해 주고 내 작업 지식창고에 모아주기를 바람.
