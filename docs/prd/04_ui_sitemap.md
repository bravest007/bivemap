# 4. UI 명세 및 사이트맵 (UI Sitemap & Specs)

[<- 마스터 인덱스로 돌아가기](./index.md)

이 문서는 코딩 전 완벽한 설계를 위해 BiveMap의 화면 구성, 전역 공통 요소(Global UX), 그리고 화면 간의 이동 흐름을 트리(Tree) 구조로 정리한 명세입니다. BiveMap은 본질적으로 화면 이동이 거의 없는 **싱글 페이지 애플리케이션(SPA)**이지만, 앱의 완전성을 위해 기능적(Modal, Panel) 화면들을 분류합니다.

---

## 1. 전역 공통 요소 (Global Common Components)
모든 화면(특히 메인 워크스페이스 내부)에서 항시 노출되거나 즉각 접근 가능한 필수 요소입니다.

- **만능 옴니 서치 & 주입바 (Top Omni-Bar)**: 화면 상단 중앙(또는 오버레이)에 상주하여 언제든 검색과 매직 복붙 업로드가 가능.
- **JARVIS 챗봇 패널 (Right Sidebar)**: 우측 공간을 차지하며 단축키나 스와이프로 부드럽게 숨김/확장이 가능한 대화창.
- **미니맵 & 포지션 트래커 (Bottom Right Minimap)**: 현재 광활한 맵 중 어디를 보고 있는지 알려주는 나침반.
- **글로벌 알림 센터 (Toast System)**: 인터넷 끊김, RAG 답변 실패, 새로운 지식 업로드 성공 시 즉각적으로 나타나는 상태 알림.

---

## 2. 화면 목록 (Screen List)

1. **`[/]` 랜딩 페이지 (Landing Screen)**: 서비스 첫 진입점. "나만의 바이브 코딩 브레인을 구축하세요" 라는 카피와 로그인 버튼 제공.
2. **`[/login]` 인증 화면 (Auth Screen)**: 이메일 또는 소셜 로그인 진입 모달.
3. **`[/app]` 메인 워크스페이스 (Main Dashboard)**: 유저가 99%의 시간을 보내는 100% 꽉 찬 무한 캔버스 마인드맵 화면.
4. **`[/app/settings]` 설정 모달 (Settings Overlay)**: 사용자 프로필, API 키 연동, 로컬 저장소 동기화 버튼, UI 테마 세팅 등.

---

## 3. 웹/모바일 반응형 동작 (Responsive Layout Shifts)
광활한 캔버스를 다루기 때문에 디바이스 크기에 따라 완벽히 다른 레이아웃 경험을 제공합니다.

- **💻 데스크탑 (Desktop View)**:
  - **스플릿 뷰(Split View)**: 좌측 중앙 70%는 마인드맵 캔버스, 우측 30%는 JARVIS 챗봇 패널이 고정 분할되어 동시에 상호작용합니다.
- **📱 모바일 / 태블릿 (Mobile View)**:
  - **100% 캔버스 + 플로팅 탭(Floating Tab)**: 화면이 좁기 때문에 100% 마인드맵 캔버스로만 채워집니다.
  - **챗봇 토글(Toggle)**: 하단에 둥둥 떠 있는 '챗봇 버튼'을 터치하면 JARVIS 패널이 바텀 시트(Bottom Sheet) 모달처럼 아래에서 위로 올라와 화면의 50%만 가리게 동작합니다. 스와이프 다운 시 다시 닫힙니다.

---

## 4. 사이트맵 (이동 흐름 트리 구조)

화면 전환은 로딩 없이 모달(Modal)과 패널(Panel) 슬라이딩 방식으로 유기적으로 이루어집니다.

```text
Root (오가닉 라우팅 트리)
│
├─ 1. [ 랜딩 페이지 (`/`) ]
│    ├─ 서비스 소개 애니메이션
│    └─ "시작하기" 버튼 클릭 
│         └─> 흐름 이동: 2. 로그인 화면으로 전환
│
├─ 2. [ 로그인/인증 화면 (`/login`) ]
│    ├─ 이메일/OAuth 로그인
│    └─ 인증 성공 시 
│         └─> 흐름 이동: 3. 메인 워크스페이스 진입
│
└─ 3. [ 메인 워크스페이스 (`/app`) ] ──⭐ (최종 목적지, Single Page Dashboard)
     │
     ├─ [ 옴니 서치 & 주입바 ] (항시 상단)
     │    ├─ 검색 수행 시 -> 마인드맵 해당 노드로 🚀초고속 자동 줌/포커스
     │    └─ URL/텍스트 복붙 시 -> AI 분석 후 맵 빈 곳에 새 노드 자동 생성 및 챗봇 RAG DB 등록
     │
     ├─ [ 인터랙티브 마인드맵 (캔버스) ] (전체 바탕화면)
     │    ├─ 휠/드래그 -> 자유로운 탐색 (패닝)
     │    ├─ 노드 클릭 & 호버 시 -> ⚡ 퀵 액션 메뉴 팝업 등장
     │    └─ 파일/이미지 드래그 앤 드롭 시 -> 지식 주입기 자동 가동 (OCR 매직 분석)
     │
     ├─ [ JARVIS 패널 ] (항시 우측)
     │    ├─ 컨텍스트 연동 -> 클릭한 마인드맵의 텍스트를 인지하여 질의응답
     │    └─ "숨기기" 클릭 -> 캔버스가 더 넓게 확장됨
     │
     └─ [ 프로필 및 설정 메뉴 ] (좌측 상단 아이콘 클릭 시)
          └─> 오버레이 모달 오픈 (`/app/settings`)
               ├─ Gemini API Keys 및 사용자 인증 정보 설정
               ├─ DB (Supabase / Local File System) 연동 스토리지 상태 확인
               └─ 다크모드 등 테마 설정
```
