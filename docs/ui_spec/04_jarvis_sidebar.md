# 4. JARVIS 사이드바 챗봇 (JARVIS Sidebar)

[<- UI 명세 마스터 인덱스로 돌아가기](./index.md)

시야의 단절 없이 광활한 마인드맵의 숲을 바라보면서 대화를 실시간으로 수행하는, 가장 중요한 RAG 전용 대화형 우측 컨트롤 패널의 컴포넌트 명세입니다.

## 4.1. 메인 패널 래퍼 (`<JarvisSidebar>`)
- **디자인 특성 철학**: 솔리드한 어두운 벽면이 아닌 극강의 글래스모피즘(`--bg-panel-glass`, `--backdrop-blur: 15px`) 패널을 씌웁니다. 패널 뒤로 우측 화면에 그려진 마인드맵 라인과 노드들이 수족관 너머처럼 은은하게 아른거려 공간이 좁아지는 답답함을 유저에게서 완전히 덜어내는 것이 목적입니다.

## 4.2. 내부 구획 - 상단 헤더부 (`<ChatTopHeader>`)
- **구조 및 컨트롤**: 상단 좌측에는 앱 브랜드/JARVIS 아바타 로고가 뜨고 우측 여백 끝에 닫기/숨기기(Collapse Toggle) 버튼 존재. 누르면 우측 화면 밖으로 깔끔하게 Panel 사라지고 캔버스가 풀 확장 동작됨. 
- **맥락 인지 UI 칩(Context Chip 활성화 상태)**: 
  - 유저가 캔버스 맵 아무 곳도 클릭 안 한 디폴트 상황에선 중앙 상단에 흐릿하게 `[전체 맵 구조 파악 상태]`가 표시됨.
  - 유저가 맵의 특정 알약 노드를 클릭하면 칩(Chip)이 파란색 발광 톤으로 1순위 변경되며 `[현재 타겟: ⚡ 특정 노드 이름]` 라고 명시적 타겟팅이 표시됨. 이후 자비스는 오직 해당 노드의 주제 안에서만 대답(RAG 스코프 좁힘). 클릭 취소 (X 버튼 처리 시) 시 전체 맥락 모드로 즉각 복귀 로직 연동.

## 4.3. 내부 구획 - 대화 히스토리 영역 (`<ChatHistoryBoard>`)
- 전형적인 iMessage 및 ChatGPT 채팅 버블 세로 스크롤 UI 설계 (`flex-direction: column`). 패널 스크롤바는 디자인에 숨김 처리.
- 사용자(User) 질문 버블은 심플한 검은색/짙은 회색 캡슐.
- JARVIS 엔진 버블의 텍스트 타이핑(출력) 효과: LLM 스트리밍이 올 텐데, 일반 챗봇들보다 체감상 `1.5배` 가량 타이머를 타이트하게 가져간 스피디(Typewriter Effect) 렌더링으로 쏘는 고성능/첨단 AI 모델 조수 인상 부여 방침 적용.
- 버블 안은 단순 텍스트가 아닌 Markdown 렌더러(`<ReactMarkdown>`) 부착으로 굵기, 색상, 그리고 특히 복사가 쉬운 `CodeBlock Prism` 코드 블럭 파티클 지원.

## 4.4. 내부 구획 - 입력 컴포넌트 (`<ChatInputBottomBox>`)
- 유저가 텍스트 명령어를 던질 수 있는 제일 밑단 박스.
- 박스 내 입력창 좌측에는 파일 첨부(`Clip` 아이콘 아이콘)가 맵핑되어 있어, 드래그&드롭을 싫어하거나 그냥 모바일에서 사진 찍은 뉴스 매체를 직접 올리고 싶을 때 PC 파일/DB 브라우저 다이어로그 호출 연동 인터페이스 포함.
