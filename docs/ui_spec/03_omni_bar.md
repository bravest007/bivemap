# 3. 만능 옴니 서치바 체계 (Omni-Bar & Ingestion)

[<- UI 명세 마스터 인덱스로 돌아가기](./index.md)

애플리케이션 화면 정중앙 상단에 위엄 있게 떠서, 유저의 텍스트 기반 입력을 판독하고 지시 체계를 모두 통제하는 '통합 입력/명령 컨트롤 타워' 컴포넌트 구조입니다.

## 3.1. 옴니 서치바 외부 체계 (`<OmniSearchBar>`)
- **UI 디자인 컴포넌트**: 
  - 긴 바(Bar) 형태의 글래스모피즘 박스(폭 최대 600px, 모바일선 90% 폭 지원). 
  - 내부에 `좌측`에는 만능형 돋보기 아이콘, `우측 끝`에는 흐릿하게 `Enter` 키보드 힌트 아이콘 노출.
  - 인풋 텍스트 컬러는 가장 가독성이 높은 밝은 솔리드 화이트, 그 외에는 투명도(`blur`).
- **인풋 판별 스위칭(Branch) 로직**: `onSubmit` 이벤트 혹은 엔터를 쳤을 때, `handleOmniSubmit(value)` 함수에서 입력값의 정규식/길이 패턴에 따라 동작을 A 혹은 B 모드로 완전히 나누어서 구동 처리합니다.

---

### [모드 A 분기] 일반 텍스트 (단어/짧은 구문) $\rightarrow$ 📌 옴니 서치(검색/이동) 모드 발동
- 입력값의 특징: 1~3개의 어절이나 "캐싱 의미", "프롬프트 종류" 등 길이가 짧고 평범한 단어들의 포진일 경우 발동.
- **동작 시퀀스 설계**: 
  1. 현재 Client 화면에 렌더링된 메모리 트리(Tree Node Array)에서 검색어와 텍스트 비교 매칭을 수행해 가장 연관성이 높은 목표 노드 ID를 찾아냄. 
  2. ReactFlow의 카메라 이동 엔진 `setCenter(x, y)` 나 `fitView()` 함수를 매끄러운 `transitionDuration(500ms)` 옵션과 함께 트리거함. 화면이 초고속 비행기나 드론처럼 화면상을 로밍(Pan/Zoom)하여 해당 목적지로 날아감.
  3. 도착한 노드는 찾아냈다는 시각적 쾌감을 주기 위해 2초 동안 `--accent-cyan` 밝은 테두리로 쿵쿵거리는 펄스(Pulse) 네온 애니메이션이 작동.

---

### [모드 B 분기] URL 또는 다량의 장문 복붙 $\rightarrow$ 📥 매직 주입기(파싱/생성) 모드 발동
- 입력값의 특징: `https://...` 등의 정규식 URL 포함 형태이나, `length` 가 수백 자 이상인 긴 프롬프트 팁 문장을 통째로 붙여넣었을 경우 발동.
- **임시 렌더링(로딩 UI/UX 변환 상태)**: 
  - 엔터를 친 직후 옴니 서치바 테두리에 무지개/오로라 그라데이션 라인이 맴도는 루핑 회전 애니메이션 작동 시킴. 
  - 이는 백그라운드 서버에서 Gemini API (파서) 가 주입된 글을 열심히 처리(RAG/Vector)하고 있다는 유저 기다림 안심 조치임.
- **완료 및 결과 생성 액션**: 
  - DB 처리가 끝나고 백엔드 파서로부터 [신규 노드 이름과 속성, 부모 카테고리 배치 좌표 x, y 값]을 결과로 Response 전달받음.
  - 브라우저 지도가 파악한 좌표 구역으로 비행해 부드럽게 이동 시킨 뒤, 텅 빈 공간 허공 속에 "뿅" 하고 새 알약 노드가 팝업 (CSS `transform: scale(0)` 에서 `1` 로 커지는 바운스 애니메이션 동반). 동시에 마인드맵 부모 가지에서 선이 부드럽게 이어진다.
  - 로딩 애니메이션 종료 및 옴니 바 내용 텍스트 리셋 포맷.
