# 3. 만능 옴니 서치바 체계 (Omni-Bar & Ingestion)

[<- UI 명세 마스터 인덱스로 돌아가기](./index.md)

애플리케이션 화면 정중앙 상단에 위엄 있게 떠서, 유저의 텍스트 기반 입력을 판독하고 지시 체계를 모두 통제하는 컨트롤 타워입니다. 기존의 "자동 탐지(Auto-detect)" 방식에서 발생할 수 있는 "원치 않는 데이터 얽힘"을 방지하기 위해 **수동 토글 UI 구조**로 전면 업그레이드 하였습니다.

## 3.1. 옴니 서치바 외형 디자인
- **UI 디자인 컴포넌트**: 
  - 긴 바(Bar) 형태의 글래스모피즘 박스(폭 최대 700px).
  - 🌟 **[핵심 신규 추가 UI] 좌측 스위치**: 사용자가 직관적으로 선택할 수 있는 작은 버튼 토글 `[🔍 Search]` 와 `[✨ Ingest]` 가 배치됩니다. 이를 통해 마인드맵에 지식을 더할 것인지, 단순히 찾을 것인지 책임을 명확히 분리합니다.
  - 내구성은 투명한 블랙홀 테마 기반, 입력 텍스트는 가독성 높은 밝은 솔리드 화이트를 채택합니다.

---

### [🔍 검색 모드 (Search Mode)] 기존 데이터 내비게이팅
- **목적**: 마인드맵 캔버스에 추가적인 쓰레기 데이터를 만들지 않고 오직 내 원래 지식(버전)들만 깔끔하게 탐색할 때 사용.
- **동작 시퀀스 설계**: 
  1. 텍스트 입력 시 DB 데이터와 `유사도/키워드`를 즉시 매칭하여 가장 핵심이 되는 노드를 타겟으로 잡음.
  2. ReactFlow의 카메라 엔진(`setViewport` / `fitView`)을 이용해 500ms 동안 시네마틱하게 줌인/팬 액션 작동.
  3. 도착 시 타겟 노드 테두리에 `--accent-cyan` 밝은 색상으로 펄스(Pulse) 네온 빛이 2초간 쿵쿵거리며 위치를 강조함.

---

### [✨ 주입 모드 (Ingest Mode)] 신규 트리 생성 및 신뢰도 검증
- **목적**: 새로운 뉴스나 긴 텍스트 기반으로 거대 지식 트리를 생성.
- **안전장치 설계 (Draft Verification)**:
  - 텍스트 덤프 후 엔터를 치면 **즉시 배경에 배치하지 않습니다.** 
  - 서치바의 회전하는 무지개 오로라 로딩 애니메이션이 끝난 직후, **임시 팝업창(Draft Modal)**이 먼저 등장.
  - 팝업창은 "이 데이터가 어디까지 신뢰할 만한 정보인가?"에 대한 AI 분석 평과 요약본을 제시.
  - 사용자가 팝업 하단의 강렬한 색상 버튼 **[Approve (맵에 주입 반영)]**을 클릭할 때만 DB API를 호출해 비로소 알약 노드들이 방사형으로 캔버스에 스폰됩니다.
